# 브리핑 우선 회의 결과물 오버라이드

이 오버라이드는 기본 회의 결과물 스킬 위에 덧붙여 적용한다. 목표는 회의 내용을 처음부터 길게 따라가게 하기보다, 읽는 사람이 `이번 회의에서 무엇이 정리됐고 다음에 무엇을 하면 되는지`를 바로 파악하게 만드는 것이다.

## 우선순위

- `회의 전체 요약` 첫 문장부터 결론, 현재 방향, 가장 중요한 변화 중 하나를 분명히 드러낸다.
- 배경 설명보다 이번 회의에서 실제로 정리된 내용, 밀린 쟁점, 바로 이어질 작업을 먼저 적는다.
- 전체 문장은 짧고 단단하게 쓰고, 읽는 사람이 30초 안에 핵심을 훑을 수 있게 압축한다.

## 섹션 작성 방식

- 섹션은 시간순 나열보다 중요도 순으로 묶는다.
- 각 섹션 첫 문장에서 해당 주제의 결론이나 현재 상태를 먼저 말한다.
- 세부 논의는 근거가 될 만큼만 남기고, 반복된 예시나 주변 맥락은 줄인다.
- 배경 설명이 꼭 필요하면 한두 문장으로만 붙이고 본론을 뒤로 미루지 않는다.

## 특히 강조할 항목

- `action_items`는 가능하면 담당자, 다음 산출물, 확인 포인트가 보이게 적는다.
- `decisions`는 확정된 내용만 짧고 단단하게 적고, 검토 중인 안건은 섞지 않는다.
- `open_questions`는 실제 후속 진행을 막는 질문 위주로만 남긴다.
- `risk_signals`는 일정 지연, 의존성, 책임 미정처럼 실행에 직접 영향을 주는 신호만 적는다.

## 피할 것

- 회의에서 길게 오간 배경 설명을 본문 대부분으로 늘어놓는 구성
- 같은 뜻의 문장을 요약, 섹션, 결정사항에 반복해서 쓰는 방식
- `논의했다`, `이야기했다`처럼 행동이나 결론이 보이지 않는 표현만 남기는 문장
- 결정되지 않은 내용을 확정된 결론처럼 과장해서 쓰는 정리
